# 后端服务启动指南

## 直接回答您的问题
**是的**，您可以使用以下命令来启动后端服务：

```bash
# 切换到backend目录（注意空格前的反斜杠）
cd /home/shopping\ web/backend

# 启动后端服务
node server.js
```

## 关于启动方式的重要说明

### 1. 前台运行 vs 后台运行

#### 前台运行（您提问的方式）
```bash
node server.js
```
- ✅ **优点**：可以实时看到输出日志
- ❌ **缺点**：断开 SSH 连接后，服务会**自动停止**

#### 后台运行（推荐方式）
```bash
# 方法1：使用nohup（推荐）
nohup node server.js > server.log 2>&1 &

# 方法2：使用screen（高级用户）
screen -S backend
node server.js
# 按 Ctrl+A+D 分离会话
```
- ✅ **优点**：断开 SSH 连接后，服务**继续运行**
- ✅ **优点**：可以查看日志文件了解运行状态

### 2. 服务管理命令

#### 检查服务是否正在运行
```bash
# 方法1：查看端口3000是否被占用
fuser 3000/tcp

# 方法2：查看node进程
ps aux | grep node
```

#### 停止服务
```bash
# 方法1：终止占用3000端口的进程
fuser -k 3000/tcp

# 方法2：终止特定的node进程
kill [进程ID]
```

#### 查看服务日志
```bash
# 前台运行时：直接看终端输出
# 后台运行时：查看日志文件
tail -f server.log  # 实时查看日志
cat server.log       # 查看所有日志
```

### 3. 验证服务是否正常

```bash
# 方法1：查看服务是否监听3000端口
netstat -tlnp | grep 3000

# 方法2：测试API接口
curl http://localhost:3000/api/products
```

## 后续建议

1. **使用后台运行**：建议使用 `nohup` 方式启动服务，确保服务持续可用
2. **定期备份**：定期备份数据库和重要配置文件
3. **监控日志**：定期查看 `server.log` 文件，及时发现问题

如果您需要手动测试或调试，可以使用前台运行方式；生产环境建议使用后台运行方式。
